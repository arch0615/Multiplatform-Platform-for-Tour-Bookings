[2.4] Mercado Pago webhook signature verification                     ~0.5 day
    Real MP signs notifications with HMAC-SHA256 of `id:<value>;ts:<ts>`
    in the `x-signature` and `x-request-id` headers.
    File: Controllers/PaymentsController.cs (Webhook action)
    - Read x-signature header (`ts=...,v1=...` format)
    - Compute HMAC-SHA256(secret, `id:{data.id};request-id:{x-request-id};ts:{ts}`)
    - Compare with v1; reject if mismatch (return 401)
    - Log all rejected signatures
    Reference: docs.mercadopago.com.mx/developers/en/guides/security/oauth-and-webhooks

================================================================================
HOW TO USE THIS SCHEDULE
================================================================================

1. At the start of each week, copy that week's tasks into a working TODO
   list. Mark them as you go.

2. Each task lists a rough effort. If a task takes 2× the estimate, stop
   and re-evaluate — usually means something upstream is missing.

3. End-of-week acceptance criteria are non-negotiable. Don't move on
   until they're met, even if it means cutting scope from later weeks.

4. Items marked "[done]" in the STATE section are baseline. Don't re-do
   them; build on them.

5. When a feature is built, add a smoke test similar to the existing
   /tmp/test-*.mjs scripts. Treat them as living regression tests.
